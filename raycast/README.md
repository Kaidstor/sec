# Sec Secrets — Raycast-расширение

Raycast-обёртка над CLI [`sec`](../cli/README.md): поиск, копирование, добавление
и генерация секретов из Raycast.

Значения секретов **не проходят через расширение**: копирование делает сам CLI
(`sec get --clip`, `sec otp --clip`, `sec gen --clip`), в UI видны только
маски, отпечатки и метаданные.

## Команды

| Команда | Что делает |
| --- | --- |
| **Search Secrets** | Список проектов/ключей (`sec ls --json`). Действия: скопировать значение (`--clip`, ⌥⌘C — с авто-очисткой буфера через 60с), TOTP-код для `kind=totp`, peek-маска (⌘P), редактирование значения/метаданных (⌘E), ссылка `proj/KEY` (⌘⇧C), отпечаток (⌘⇧F), история версий с undo/redo (⌘H), удаление (⌃X). |
| **Add Secret** | Форма нового секрета. Значение уходит в `sec set --stdin` (не в argv), с повтором ввода как в CLI. Поддерживает `--note`, `--kind` и инстансы (`proj@env`). |
| **Generate Secret** | `sec gen` — случайное значение нужной длины, опционально `--symbols` и сразу в буфер. |

## Установка (локально)

```sh
cd raycast
npm install
npm run dev   # импортирует расширение в Raycast; после Ctrl+C оно остаётся
```

Требуется установленный `sec` (`brew install kaidstor/tap/sec`). Путь к бинарю
ищется в `/opt/homebrew/bin`, `/usr/local/bin`, `~/go/bin`; другой путь можно
задать в настройках расширения (Sec Binary Path).

## Заметки

- Инстансы (`service@env`) показываются как отдельные секции; при вызовах CLI
  ref режется на `service/KEY -e env` (в ref символ `@` не допускается).
- Первый вызов из Raycast может показать системный диалог Keychain — мастер-ключ
  хранилища лежит в macOS Keychain, разреши доступ («Always Allow»).
- Расширение локальное, в Raycast Store не публикуется.
